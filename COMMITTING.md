# Committing OMP Learner

This guidance applies only to `klondikemarlen/omp-learner`; shared commit guidance remains authoritative elsewhere.

Keep each cohesive feature in one commit: implementation, focused tests, relevant documentation, and the required package-version update belong together when they deliver one observable behavior.

Keep unrelated dependency churn, formatting, migrations, and workflow-learning edits in separate commits. Split unrelated features even when they touch the same files.
